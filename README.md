GitHub-hosted LaTeX templates for [LATIUM](https://github.com/Ogro-Projukti/lat-mini-private). The app fetches `registry.json` and individual template files via a backend proxy.

## Structure

```
registry.json                 # Generated by scripts/build_registry.py
scripts/
  build_registry.py           # Builds registry.json from templates/*/meta.json
  remove_template.py          # Removes templates/{id}/ and regenerates catalog
  badge-templates.json        # Shields.io badge (template count)
  badge-updated.json          # Shields.io badge (registry date)
templates/
  {template-id}/
    meta.json                 # Contributor + maintainer metadata (source of truth)
    main.tex
    preview.png               # Optional, 300×400
  ...
```

## For contributors

Open a [**Submit a Template**](https://github.com/Ogro-Projukti/latex-templates/issues/new?template=submit_template.md) issue with a public folder link and a valid `meta.json` at the folder root.

## For maintainers

See **[MAINTAINER.md](MAINTAINER.md)** for the full issue → publish workflow.

After adding or updating a template folder, regenerate the catalog:

```bash
python scripts/build_registry.py
git add templates/ registry.json scripts/badge-*.json
git commit -m "Add template: <name>"
git push
```

Use `--check` in CI or before opening a PR:

```bash
python scripts/build_registry.py --check
```

To remove a template by id:

```bash
python scripts/remove_template.py <template-id> --dry-run
python scripts/remove_template.py <template-id>
```

## Publishing to LATIUM

1. Templates live on [Ogro-Projukti/latex-templates](https://github.com/Ogro-Projukti/latex-templates) `main`.
2. Verify raw URLs work:
   ```bash
   curl -s "https://raw.githubusercontent.com/Ogro-Projukti/latex-templates/main/registry.json"
   curl -s "https://raw.githubusercontent.com/Ogro-Projukti/latex-templates/main/templates/article-starter/main.tex"
   ```
3. LATIUM settings (`latium/settings.py` or env):
   - `TEMPLATE_REGISTRY_URL`
   - `TEMPLATE_RAW_URL_PREFIXES`

## Preview images

Add `preview.png` (≈300×400), set `"preview": true` and `"previewImagePath": "preview.png"` in `meta.json`, then run `build_registry.py`. Until then, the app shows a placeholder graphic.

## Legacy note

The main LATIUM repo also has `backend/latex/templates/generate_registry.py` for bootstrapping this library from built-in templates. Day-to-day publishing in **this** repo uses `scripts/build_registry.py` only.
